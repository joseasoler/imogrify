# Contributing to imogrify

Thank you for your interest on contributing!

## Questions and bug reports

You can use the [issue tracker](ToDo) to ask questions and report bugs. Please use the search (including closed issues) to check for previous discussions.

## Contributions

### Source code contributions

Feature proposals should be discussed on the [issue tracker](ToDo) before creating a pull request, to allow for community discussion.

Pull requests should describe the reasoning behind their implementation.

dev- Cmake presets run [clang-tidy](https://clang.llvm.org/extra/clang-tidy) and [clang-format](https://clang.llvm.org/docs/ClangFormat.html) automatically. You will need recent versions of these tools. Unit tests must also run successfully.

imogrify uses UTF-8 encoding by default. Check the [UTF-8 Everywhere Manifesto](http://utf8everywhere.org) for details.
